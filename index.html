How much do you like to write unit tests from the scratch?

Most of the answer would be “not so much”. The reason behind this guess is that writing the first successful unit test is just a copy, paste, and modification job, and it's not a pleasant task.

What if there is a command that can do this job (copy, paste, and modification)?

TL;DR; Yes, there is a command.

Let's say you coded your Angular component, and you want to write a unit test for the component, The following command will auto-generated the unit test for it, and it works like a charm. 

        $ npm install ngentest -g # to run this command anywhere
        $ gentest my.component.ts # node_modules/.bin/gentest

How is it possible? Because the command will parse the following data from your Typescript class.

  • Class Name
  • Imports
  • Inputs, Outputs
  • Mocks required
  • Providers for TestBed
  • List of functions to test

By running a ngentest command, more than half of your test will be done. It actually generates much more than half of your unit test. With this help, you can save a lot of time. These are the minimal coverage percentages number by ngentest for each type. For simple components and directives, it easily covers 100% of your code.

  • Component (65%)
  • Directive (65%)
  • Service (70%)
  • Pipe (80%)
